foreword

the purpose of this lib is to provide a minimal engine to display a TMS (Tile Map Service) like OSM, MapBox, MapQuest... there are already many APIs out there such as leaflet and they work fine, they are very comprehensive, well tested so I'd recommend them for a mainstream use.
I found it impractical to work with for "advanced" use cases ; I needed to draw the view in a canvas (something leaflet wasn't designed for) and had to hack my way so much so that I ended up rebuilding it from scratch.

this means that it's rough around the edges, not fully tested and has lots of room for improvement. in a word, not for the faint of the heart.

basics

this is how you instantiate light map: 

//import the script "light-map.min.js" somewhere in your page
//then:
var provider = "http://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png";
var domains = "a,b,c".split( ',' );
var map = new Map( provider, domains, 512,512,0,18 );
map.setView(0,0,1);
which gives you this the above is an actual map, loaded from the OpenStreetMap TMS.

the constructor takes 6 arguments which are respectively:

  • provider: the urls of the TMS
  • domains: an array of domains to fetch images from
  • width: the width of the view rect
  • height: the height of the view rect
  • minZoom: the minimum zoom level
  • maxZoom: the maximum zoom level

now try to push this button (& allow geolocation)


the button calls this method; 
function goHome()
{
    var geoSuccess = function(position) {

        map.latitude  = position.coords.latitude;
        map.longitude = position.coords.longitude;
        map.zoom = 17;
        map.setView();
    };
    navigator.geolocation.getCurrentPosition(geoSuccess);
}
here map.setView() is called without arguments, because map.latitude, map.longitude and map.zoom values are set before the call.
usually, you'll use setView( lat, lon, zoom) where lat & lon are expressed in degrees and zoom is a zoom value comprised between the bounds you specified in the constructor
